# 核心碰撞算法技术文档

## 1. 概述

### 1.1 背景

在可信联网搜索系统中，当多个信源对同一事件或事实进行报道时，往往会出现信息冲突的情况。这种冲突被称为**碰撞（Collision）**。例如：
- 不同信源对同一事件的数据描述不一致
- 不同信源对同一事件的定性结论相反
- 不同信源引用的来源或原因存在差异

核心碰撞算法旨在自动检测这些碰撞，并通过优先级规则智能裁决，最终输出最可信的信息结论。

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| **SearchResult** | 搜索引擎返回的单一结果，包含标题、URL、摘要等信息 |
| **Claim** | 从 SearchResult 中提取的核心陈述，代表该结果的主要观点 |
| **FactCollision** | 检测到的碰撞事件，包含多个相互冲突的 Claims |
| **PriorityLevel** | 优先级层级枚举，定义不同类型信源的优先级顺序 |
| **ConsensusDegree** | 共识度，表示碰撞解决后最高优先级 Claim 的占比 |

### 1.3 算法流程概览

```
输入: List[SearchResult]
        │
        ▼
┌───────────────────┐
│ 1. Claim 提取      │  从每个 SearchResult 提取 Claim
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 2. 碰撞分组        │  基于相似度将相似的 Claims 分组
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 3. 优先级裁决      │  按优先级规则确定胜出 Claim
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 4. 共识度计算      │  计算裁决结果的共识程度
└───────────────────┘
        │
        ▼
   输出: List[FactCollision]
```

## 2. Claim 提取算法

### 2.1 数据结构

Claim 的数据结构定义如下（参考 `src/web_search/core/models.py`）：

```python
@dataclass
class Claim:
    result: SearchResult       # 来源搜索结果
    statement: str            # 核心陈述文本
    key_facts: List[str]      # 关键事实点列表
    timestamp: datetime       # 提取时间戳
```

### 2.2 提取规则

Claim 提取由 `ClaimExtractor` 类实现（参考 `src/web_search/resolver/claim.py`）。

**提取算法伪代码：**

```
function extract(result: SearchResult) -> List[Claim]:
    1. statement = result.snippet 前 200 字符
    2. if statement 为空:
           statement = result.title
    3. key_facts = _extract_key_facts(result)
    4. return [Claim(result, statement, key_facts, 当前时间)]
```

**实现细节：**

```python
def extract(self, result: SearchResult) -> List[Claim]:
    """从搜索结果中提取声明"""
    statement = result.snippet[:200] if result.snippet else ""
    if not statement:
        statement = result.title

    claim = Claim(
        result=result,
        statement=statement,
        key_facts=self._extract_key_facts(result),
        timestamp=datetime.now()
    )
    return [claim]
```

### 2.3 关键事实点提取

关键事实点提取算法：

```
function _extract_key_facts(result: SearchResult) -> List[str]:
    facts = []
    1. if result.title 非空:
           facts.append(result.title)
    2. if result.snippet 非空:
           facts.append(result.snippet 前 100 字符)
    3. return facts
```

**当前实现为简化版本**，仅提取标题和摘要前100字符作为关键事实点。理想实现应包含：
- 命名实体识别（人名、地名、机构名）
- 数值提取（日期、百分比、数量）
- 事件动作提取

## 3. 相似度计算算法

### 3.1 算法选择

当前系统采用 **Jaccard 相似度** 算法计算两个 Claim 之间的相似程度。

Jaccard 相似度是一种集合相似度度量方法，计算两个集合的交集与并集之比：

```
Jaccard(A, B) = |A ∩ B| / |A ∪ B|
```

### 3.2 Jaccard 相似度公式

对于两个 Claim 语句：

```
similarity = |words(claim1) ∩ words(claim2)| / |words(claim1) ∪ words(claim2)|
```

其中 `words(claim)` 表示将 claim 语句按空格分词后的小写词语集合。

**实现代码（参考 `src/web_search/resolver/fact_resolver.py`）：**

```python
def _are_claims_similar(self, c1: Claim, c2: Claim) -> bool:
    """判断两个claim是否描述同一事实"""
    words1 = set(c1.statement.lower().split())
    words2 = set(c2.statement.lower().split())
    overlap = len(words1 & words2) / len(words1 | words2)
    return overlap > 0.6
```

### 3.3 阈值设定

当前阈值为 **0.6**，即当两个 claim 的 Jaccard 相似度超过 0.6 时，认为它们描述的是同一事实，可以归入同一碰撞组。

**阈值选择依据：**
- 阈值过高（如 0.8）会导致过度严格，相似表述被误判为不相似
- 阈值过低（如 0.4）会导致过度宽松，不同事实被误判为相似
- 0.6 是文本相似度任务的常见经验阈值

## 4. 碰撞分组算法

### 4.1 分组策略

碰撞分组采用 **贪心最近邻聚类** 算法，将相似的 Claims 自动聚合到同一组：

```
function _group_similar_claims(claims: List[Claim]) -> List[List[Claim]]:
    groups = []
    used = set()

    for i, claim in enumerate(claims):
        if i in used:
            continue

        group = [claim]
        used.add(i)

        for j, other in enumerate(claims[i+1:], start=i+1):
            if j in used:
                continue
            if _are_claims_similar(claim, other):
                group.append(other)
                used.add(j)

        groups.append(group)

    return groups
```

### 4.2 算法特点

- **贪心策略**：以第一个未处理 claim 为组心，遍历后续所有 claim
- **单向扩展**：每个 claim 只会被分配到一个组，不会出现跨组情况
- **时间复杂度**：O(n²)，其中 n 为 claim 总数

### 4.3 流程图

## 5. 优先级裁决算法

### 5.1 优先级层级定义

系统定义了 10 个优先级层级（参考 `src/web_search/resolver/priority_engine.py`）：

```python
class PriorityLevel(IntEnum):
    BLACK_INDIVIDUAL = 1      # 黑名单-个人
    BLACK_MEDIA = 2          # 黑名单-媒体
    GRAY_INDIVIDUAL = 3      # 灰名单-个人
    GRAY_MEDIA = 4           # 灰名单-媒体
    GRAY_KOL_SMALL = 5       # 灰名单-KOL(小, 粉丝>=1万)
    GRAY_KOL_MEDIUM = 6      # 灰名单-KOL(中, 粉丝>=10万)
    GRAY_KOL_BIG = 7         # 灰名单-KOL(大, 粉丝>=100万)
    WHITE_MUNICIPAL = 8      # 白名单-市级机构
    WHITE_PROVINCIAL = 9      # 白名单-省级机构
    WHITE_NATIONAL_OFFICIAL = 10  # 白名单-国家级机构
```

### 5.2 优先级层级示意

```

Level 8:  WHITE_MUNICIPAL     ████████
Level 9:  WHITE_PROVINCIAL    █████████
Level 10: WHITE_NATIONAL       ██████████

颜色图例:
█ 黑名单 (Black List)
█ 灰名单 (Gray List)
█ 白名单 (White List)
```

### 5.3 裁决规则

优先级计算函数 `calculate_priority()` 的裁决规则：

```
输入: SearchResult
输出: (PriorityLevel, rule_description)

规则顺序:
1. if classification == BLACK:
      if source_type == INDIVIDUAL:
          return BLACK_INDIVIDUAL
      else:
          return BLACK_MEDIA

2. if classification == GRAY:
      if source_type == KOL:
          if 粉丝数 >= 1000000:
              return GRAY_KOL_BIG
          elif 粉丝数 >= 100000:
              return GRAY_KOL_MEDIUM
          else:
              return GRAY_KOL_SMALL
      else if source_type == INDIVIDUAL:

      else:

      if source_level == NATIONAL:

          return WHITE_PROVINCIAL
      elif source_level == MUNICIPAL:
          return WHITE_MUNICIPAL

```

## 6. 共识度计算模型

### 6.1 定义

**共识度（Consensus Degree）** 衡量碰撞解决结果的可信程度，定义为：在同一碰撞组中，与最高优先级 Claim 拥有相同优先级的 Claim 数量占总 Claim 数量的比例。

### 6.2 计算公式

```
consensus_degree = top_priority_count / total_count
```

其中：
- `top_priority_count`：与最高优先级 Claim 优先级相同的 Claim 数量
- `total_count`：碰撞组中 Claim 的总数量

### 6.3 计算示例

假设某碰撞组包含 5 个 Claims，其优先级分布如下：

| Claim | PriorityLevel | 描述 |
|-------|--------------|------|
| A | WHITE_NATIONAL_OFFICIAL (10) | 白名单-国家级 |
| B | WHITE_NATIONAL_OFFICIAL (10) | 白名单-国家级 |
| C | WHITE_PROVINCIAL (9) | 白名单-省级 |
| D | GRAY_MEDIA (4) | 灰名单-媒体 |
| E | GRAY_INDIVIDUAL (3) | 灰名单-个人 |

计算过程：
1. 最高优先级 = 10（WHITE_NATIONAL_OFFICIAL）
2. 最高优先级 Claim 数量 = 2（A 和 B）
3. 共识度 = 2 / 5 = 0.4

### 6.4 实现代码

```python
def resolve_collision(self, claims: List[Claim]) -> FactCollision:
    """解决碰撞"""
    if len(claims) < 2:
        return FactCollision(
            collision_id="single",
            claims=claims,
            resolved_claim=claims[0] if claims else None,
            priority_rule_used="single_claim",
            consensus_degree=1.0
        )

    # 计算每个claim的优先级
    scored_claims = []
    for claim in claims:
        priority, rule = self.calculate_priority(claim.result)
        scored_claims.append((claim, priority, rule))

    # 按优先级排序（降序）
    scored_claims.sort(key=lambda x: x[1], reverse=True)

    # 计算共识度
    top_priority = scored_claims[0][1]
    top_count = sum(1 for _, p, _ in scored_claims if p == top_priority)
    consensus = top_count / len(claims)

    return FactCollision(
        collision_id=self._generate_collision_id(claims),
        claims=claims,
        resolved_claim=scored_claims[0][0],
        priority_rule_used=scored_claims[0][2],
        consensus_degree=consensus
    )
```

## 7. 算法复杂度分析

### 7.1 时间复杂度

整体碰撞检测与解决流程的时间复杂度：

| 阶段 | 时间复杂度 | 说明 |
|------|----------|------|
| Claim 提取 | O(n) | 每个 SearchResult 处理一次 |
| 相似度计算 | O(n²) | 每对 Claims 计算一次 Jaccard |
| 碰撞分组 | O(n²) | 分组过程中每对 Claims 比较一次 |
| 优先级裁决 | O(n) | 每个 Claim 计算一次优先级 |

**总体时间复杂度：O(n²)**

其中 n 为输入的 SearchResult 总数。

### 7.2 空间复杂度

| 数据结构 | 空间复杂度 | 说明 |
|----------|----------|------|
| Claims 列表 | O(n) | 存储 n 个 Claim 对象 |
| 分组结果 | O(n) | 最多 n 个组，每组至少1个 Claim |
| 优先级计算中间变量 | O(n) | scored_claims 列表 |

**总体空间复杂度：O(n)**

### 7.3 性能估算

假设单次搜索返回 10 条结果：

| 阶段 | 操作次数 |
|------|---------|
| Claim 提取 | 10 次 |
| 相似度计算 | 45 次（10×9/2） |
| 优先级裁决 | 10 次 |

实际性能影响可忽略不计。

## 8. 当前局限性与改进方向

### 8.1 当前局限性

| 局限性 | 说明 | 影响 |
|--------|------|------|
| **简单分词** | 使用空格分词，未处理中文分词 | 中文场景下相似度计算不准确 |
| **Jaccard 局限** | 仅考虑词汇重合，不考虑语义 | 语义相似但用词不同的 Claims 可能被漏判 |
| **贪心分组** | 可能陷入局部最优 | 同一事实的多条表述可能被分到不同组 |
| **单一阈值** | 所有场景使用 0.6 阈值 | 无法适应不同类型内容的特性 |
| **无增量更新** | 每次都重新计算全部 | 实时系统中无法高效更新 |

### 8.2 改进方向

#### 8.2.1 语义相似度计算

**改进方案：** 引入文本嵌入（Embedding）模型计算语义相似度

```
改进前: Jaccard(word_set1, word_set2)
改进后: CosineSimilarity(Embedding(claim1), Embedding(claim2))
```

**优势：**
- 捕捉语义相似性而非词汇重叠
- 更好地处理同义词、表述差异

#### 8.2.2 中文分词处理

**改进方案：** 使用中文分词工具（如 jieba）替代空格分词

```python
import jieba
words1 = set(jieba.cut(c1.statement.lower()))
words2 = set(jieba.cut(c2.statement.lower()))
```

#### 8.2.3 层次聚类算法

**改进方案：** 替代贪心分组，使用层次聚类（Agglomerative Clustering）

**优势：**
- 自动确定最优分组数量
- 避免贪心策略的局部最优问题

#### 8.2.4 自适应阈值

**改进方案：** 根据内容类型动态调整阈值

```python
def _are_claims_similar(self, c1: Claim, c2: Claim) -> bool:
    base_threshold = 0.6
    # 短文本提高阈值要求
    if len(c1.statement) < 50 or len(c2.statement) < 50:
        return overlap > 0.75
    # 长文本可适当降低
    if len(c1.statement) > 200 and len(c2.statement) > 200:
        return overlap > 0.5
    return overlap > base_threshold
```

#### 8.2.5 专业领域适配

**改进方案：** 针对不同领域（财经、医疗、法律）训练专门的分类模型或调整权重
