## 1. 语义冲突识别：从“字面重合”转向“逻辑推断”

目前的 Jaccard 相似度只能判断“大家是不是在聊同一件事”，却无法识别“大家说的是不是一回事”。

* **引入 NLI (Natural Language Inference) 模型：**
    单纯的相似度无法区分对抗性陈述。例如：“A公司**已**收购B公司”与“A公司**未**收购B公司”，其 Jaccard 或 Embedding 相似度都极高，但逻辑上是**冲突（Contradiction）**。
    * **方案：** 在碰撞分组后，利用 Cross-Encoder 架构的 NLI 模型对组内 Claims 进行两两比对，标记为 `Entailment` (蕴含)、`Neutral` (中立) 或 `Contradiction` (矛盾)。
* **多维向量空间 (Hybrid Search)：**
    不要只依赖分词，建议引入基于预训练模型（如 BERT, RoBERTa 或专为搜索优化的 BGE 向量）的余弦相似度。
    
---

## 2. 事实颗粒度优化：从“文本片段”转向“结构化三元组”

目前的 Claim 提取太粗糙（摘要前 200 字），这会导致大量噪音进入碰撞池。

* **SPO 三元组提取：**
    利用 LLM 或轻量级 NLP 工具提取 **Subject (主体) - Predicate (谓词) - Object (客体)**。
    * *例子：* “伊朗 (S) 攻击 (P) 以色列 (O)”。
    * **碰撞逻辑：** 当两个 Claim 的 S 和 O 相同，但 P 语义相反（或数值不同）时，触发核心碰撞警告。
* **数值与时间敏感：**
    针对财经或灾难数据，碰撞往往发生在**数字**上。
    * **策略：** 强制提取文本中的数字、百分比和日期。如果两个 Claim 的语义相似度高但数字差异超过阈值（如 $10\%$ 以上），则判定为事实碰撞。

---

## 3. 动态优先级与可信度模型

目前的优先级是静态的“白/灰/黑”名单，缺乏对信息时效性和信源行为的考量。

* **时效性衰减 (Time Decay)：**
    在联网搜索中，**“新”**往往比**“权威”**更重要（尤其是突发事件）。
    $$FinalPriority = BasePriority \cdot e^{-\lambda(t_{now} - t_{publish})}$$
    * 其中 $\lambda$ 是衰减系数，确保旧的官方通报在新的现场快报面前有一定的裁决修正。
* **多样性加权 (Diversity Weighting)：**
    如果 5 个相互独立的信源（非互相转载）指向同一个结论，即使它们的单体优先级低于一个省级媒体，群体的共识度也应赋予更高的权重。

---

## 4. 冲突解释性：从“输出结果”转向“输出理由”

一个可信的搜索系统不仅要告诉用户结论，还要告诉用户**为什么存在冲突**。

* **碰撞类型标记：**
    在 `FactCollision` 数据结构中增加 `collision_type` 字段：
    * `OUTDATED`：信息因时间产生偏差（如：旧房价 vs 新房价）。
    * `PERSPECTIVE`：立场不同导致的描述差异。
    * `FACTUAL_ERROR`：明显的谣言或数据错误。
* **共识度算法修正：**
    目前的共识度计算过于简单。建议引入**“异议强度”**概念：如果组内最高优先级是 Level 10，但存在多个 Level 9 的信源持相反意见，共识度应大幅下降，并触发“争议中”标签。

---

## 5. 架构改进建议：引入“双层聚类”

为了平衡 O(n²) 的计算压力：

1.  **粗聚类 (Coarse Clustering)：** 使用高效率的向量检索 (Vector DB) 或词频过滤，将海量结果快速划分到不同的“事件桶”中。
2.  **精碰撞 (Fine-grained Collision)：** 在每个小桶内（通常 < 20 个结果），调用 LLM 或 NLI 模型进行深度的逻辑对撞。

---

### 改进后的 FactCollision 数据结构示意

```python
@dataclass
class EnhancedFactCollision:
    collision_id: str
    primary_fact: Claim            # 裁决出的主要事实
    conflicting_claims: List[Claim] # 明确冲突的声明
    supporting_claims: List[Claim]  # 支持主要事实的声明
    contradiction_score: float      # 冲突剧烈程度 (0-1)
    reasoning: str                  # 裁决理由（例如："由于新华社发布时间更晚且属于国家级媒体，覆盖了早期社交媒体的传闻"）
```
