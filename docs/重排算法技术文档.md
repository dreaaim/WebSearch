# 重排（Reranker）算法技术文档

## 1. 概述

重排（Reranker）模块位于可信联网搜索流程的最终阶段，负责对经过筛选、分类和碰撞裁决的搜索结果进行综合排序，输出最终的高可信度结果列表。

### 1.1 模块位置

```
用户查询 ──→ 改写 ──→ 搜索 ──→ LLM筛选 ──→ Embedding碰撞 ──→ LLM裁决 ──→ 重排 ──→ 返回
                                                                         ▲
                                                                         │
                                                                    [Reranker]
```

### 1.2 核心职责

- **综合评分**: 融合相关性、可信度、时效性、权威性等多个维度计算综合得分
- **外部重排**: 支持调用外部专业重排模型（如 Cohere、Jina、阿里 DashScope 等）获取额外重排分数
- **Top-K 筛选**: 从所有候选结果中选取得分最高的 K 条结果
- **碰撞裁决整合**: 将 LLM 碰撞裁决结果纳入评分体系

## 3. 综合评分算法

### 3.1 评分因子

Reranker 综合以下五个因子计算最终得分：

| 因子 | 键名 | 默认权重 | 说明 |
|------|------|----------|------|
| 相关性 | `relevance` | 0.30 | 内容与查询的相关程度 |
| 可信度 | `trustworthiness` | 0.20 | 信源的黑白名单分类 |
| 时效性 | `freshness` | 0.10 | 内容发布时间的衰减评分 |
| 权威性 | `authority` | 0.10 | 信源的类型和级别 |
| 外部重排 | `external` | 0.30 | 外部重排模型的评分 |

### 3.2 评分公式

#### 3.2.1 无外部重排分数时

```
FinalScore = w_relevance × RelevanceScore
           + w_trustworthiness × TrustworthinessScore
           + w_freshness × FreshnessScore
           + w_authority × AuthorityScore
           + JudgmentBonus
```

#### 3.2.2 有外部重排分数时

```
FinalScore = (1 - w_external) × InternalScore
           + w_external × ExternalScore
```

其中：

```
InternalScore = w_relevance × RelevanceScore
              + w_trustworthiness × TrustworthinessScore
              + w_freshness × FreshnessScore
              + w_authority × AuthorityScore
              + JudgmentBonus
```

### 3.3 各因子详细计算

#### 3.3.1 相关性得分（RelevanceScore）

```python
def _get_relevance(self, result: Any) -> float:
    if hasattr(result, 'relevance_score'):
        return result.relevance_score
    if hasattr(result, 'result') and hasattr(result.result, 'relevance_score'):
        return result.result.relevance_score
    return 0.5  # 默认中间值
```

- 来源：优先取结果的 `relevance_score` 属性或嵌套的 `result.relevance_score`
- 原始分值范围：0-10
- 归一化后：0-1.0（除以 10.0）

#### 3.3.2 可信度得分（TrustworthinessScore）

```python
def _get_trustworthiness(self, result: Any) -> float:
    if hasattr(result, 'is_blacklisted') and result.is_blacklisted:
        return 0.0

    classification = self._get_classification(result)
    if classification == Classification.WHITE:
        return 1.0
    if classification == Classification.GRAY:
        return 0.5
    return 0.3  # INDIVIDUAL 等默认 0.3
```

| Classification | 得分 |
|----------------|------|
| BLACK (被黑名单) | 0.0 |
| WHITE | 1.0 |
| GRAY | 0.5 |
| 其他 | 0.3 |

#### 3.3.4 权威性得分（AuthorityScore）

```python
def _get_authority_score(self, result: Any) -> float:
    source_type = self._get_source_type(result)
    source_level = self._get_source_level(result)

    if source_type == SourceType.OFFICIAL:
        level_scores = {
            SourceLevel.NATIONAL: 1.0,
            SourceLevel.PROVINCIAL: 0.85,
            SourceLevel.MUNICIPAL: 0.7,
            SourceLevel.LOCAL: 0.5
        }
        return level_scores.get(source_level, 0.5)

    if source_type == SourceType.MEDIA:
        level_scores = {
            SourceLevel.NATIONAL: 0.8,
            SourceLevel.PROVINCIAL: 0.65,
            SourceLevel.MUNICIPAL: 0.5,
            SourceLevel.LOCAL: 0.35
        }
        return level_scores.get(source_level, 0.35)

    if source_type == SourceType.KOL:
        return 0.5  # KOL_MEDIUM

    return 0.3  # INDIVIDUAL
```

| SourceType | NATIONAL | PROVINCIAL | MUNICIPAL | LOCAL |

| OFFICIAL | 1.0 | 0.85 | 0.7 | 0.5 |
| MEDIA | 0.8 | 0.65 | 0.5 | 0.35 |
| KOL | 0.5 | 0.5 | 0.5 | 0.5 |
| INDIVIDUAL | 0.3 | 0.3 | 0.3 | 0.3 |

## 4. 时效性评分算法（FreshnessCalculator）

### 4.1 算法设计

FreshnessCalculator 采用**分段阶梯衰减**策略，根据内容发布时间与当前时间的距离远近，赋予不同的时效性得分。

### 4.2 时间分段

| 时间区间 | 天数范围 | 默认得分 |
|----------|----------|----------|
| 7 天内 | 0 ≤ days ≤ 7 | 1.0 |
| 30 天内 | 8 ≤ days ≤ 30 | 0.8 |
| 90 天内 | 31 ≤ days ≤ 90 | 0.6 |
| 1 年内 | 91 ≤ days ≤ 365 | 0.4 |
| 更久 | days > 365 | 0.2 |
| 无日期信息 | - | 0.5 |

### 4.3 实现代码

```python
class FreshnessCalculator:
    def __init__(
        self,
        period_7d_score: float = 1.0,
        period_30d_score: float = 0.8,
        period_90d_score: float = 0.6,
        period_1y_score: float = 0.4,
        older_score: float = 0.2
    ):
        self.period_7d_score = period_7d_score
        self.period_30d_score = period_30d_score
        self.period_90d_score = period_90d_score
        self.period_1y_score = period_1y_score
        self.older_score = older_score

    def calculate(self, published_date: Optional[str]) -> float:
        if not published_date:
            return 0.5  # 无日期信息给予中间值

        try:
            pub_date = datetime.fromisoformat(published_date.replace("Z", "+00:00"))
            days_old = (datetime.now() - pub_date.replace(tzinfo=None)).days

            if days_old <= 7:
                return self.period_7d_score
            elif days_old <= 30:
                return self.period_30d_score
            elif days_old <= 90:
                return self.period_90d_score
            elif days_old <= 365:
                return self.period_1y_score
            else:
                return self.older_score
        except:
            return 0.5  # 解析失败给予中间值
```

### 4.4 时间复杂度

- **时间复杂度**: O(1) — 仅需计算天数差并做常数次比较
- **空间复杂度**: O(1) — 无额外存储

### 4.5 衰减曲线示意

```

0.8 |      ████████
0.6 |            ████████
0.4 |                  ████████
```

## 6. 重排流程

### 6.1 同步重排流程（rerank）

```python
def rerank(
    self,
    results: List[Any],
    judgments: List = None,
    top_k: int = None
) -> List[Any]:
    top_k = top_k or self.config.top_k

    scored_results = []
    for result in results:
        score = self._calculate_composite_score(result, judgments)
        scored_results.append((result, score))

    # 按综合得分降序排序
    scored_results.sort(key=lambda x: x[1], reverse=True)

    ranked = [result for result, _ in scored_results]
    return ranked[:top_k] if top_k > 0 else ranked
```

**步骤：**
1. 遍历所有候选结果
2. 对每个结果调用 `_calculate_composite_score` 计算综合得分
3. 按得分降序排序
4. 返回前 top_k 条结果

### 6.2 异步重排流程（rerank_async）

```python
async def rerank_async(
    self,
    results: List[Any],
    query: str,
    judgments: List = None,
    top_k: int = None,
    original_query: str = None,
    search_query: str = None
) -> List[Any]:
    top_k = top_k or self.config.top_k

    if self.reranker_client and results:
        # 1. 提取文本
        texts = []
        for r in results:
            if hasattr(r, 'result') and hasattr(r.result, 'snippet'):
                texts.append(r.result.snippet or r.result.title)
            elif hasattr(r, 'snippet'):
                texts.append(r.snippet or r.title)
            else:
                texts.append(str(r))

        # 2. 调用外部重排服务
        rerank_results = await self.reranker_client.rerank(
            query=query,
            texts=texts,
            top_n=len(results),
            search_query=search_query,
            instructions=original_query
        )

        # 3. 将外部分数写入结果
        self._external_scores = {r.index: r.score for r in rerank_results}
        for i, result in enumerate(results):
            if i in self._external_scores:
                if hasattr(result, 'external_rerank_score'):
                    result.external_rerank_score = self._external_scores[i]
                elif hasattr(result, 'relevance_score'):
                    result.relevance_score = self._external_scores[i]

    # 4. 执行内部重排
    return self.rerank(results, judgments, top_k)
```

**步骤：**
1. 调用外部重排服务获取外部分数
2. 将外部分数注入结果对象
3. 调用内部 `rerank` 执行综合排序

## 8. 数据流与类型

### 8.1 输入数据类型

Reranker 接受任意类型的 `results` 列表，通过属性访问协议获取评分所需字段：

| 属性路径 | 类型 | 说明 | 获取优先级 |
|----------|------|------|------------|
| `relevance_score` | float | 相关性得分 | 最高 |
| `result.relevance_score` | float | 嵌套相关性得分 | 次高 |
| `is_blacklisted` | bool | 是否被黑名单 | - |
| `classification` | Classification | 黑白灰分类 | - |
| `source_info.source_type` | SourceType | 信源类型 | - |
| `source_info.source_level` | SourceLevel | 信源级别 | - |
| `result.published_date` | str | 发布时间 (ISO 格式) | - |
| `published_date` | str | 发布时间 (ISO 格式) | - |
| `external_rerank_score` | float | 外部重排得分 | - |
| `result.external_rerank_score` | float | 嵌套外部重排得分 | - |

### 8.2 输出数据

返回类型同输入类型，经排序后按序输出。

## 9. 关键设计决策

### 9.1 为什么采用加权求和而非机器学习模型？

- **可解释性**: 每个因子的权重直观可调，便于人工干预和调优
- **实时性**: O(n) 时间复杂度，适合在线服务
- **可控性**: 可通过调整权重快速适配不同业务场景

### 9.2 为什么引入外部重排服务？

- **语义理解**: 外部重排模型（如 Cohere rerank）基于深度学习模型，能更好理解查询与文档的语义匹配
- **解耦设计**: 内部重排负责业务规则（权威性、时效性），外部重排负责语义相关性
- **混合评分**: 通过 `external_rerank_weight` 灵活控制两者比例

### 9.3 时效性为何采用阶梯衰减而非指数衰减？

- **业务匹配**: 新闻类内容通常以天为单位有显著差异，阶梯衰减更符合内容生命周期规律
- **调优友好**: 各区间得分可通过配置独立调整
- **边界清晰**: 用户可明确知道"7天内"得分为 1.0，便于预期
