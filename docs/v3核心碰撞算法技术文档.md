# v3 核心碰撞算法技术文档

## 1. 概述

### 1.1 背景

在可信联网搜索系统中，当多个信源对同一事件或事实进行报道时，往往会出现信息冲突的情况。这种冲突被称为**碰撞（Collision）**。v3架构在v1/v2基础上引入了**正交碰撞检测（Orthogonal Collision Detection）**机制，从NLI语义、SPO三元组、数值/时间三个维度进行正交检测，大幅提升了碰撞检测的准确性。

### 1.2 v1/v2/v3 碰撞算法演进对比

| 维度 | v1 架构 | v2 架构 | v3 架构 |
|------|---------|---------|---------|
| **碰撞检测方式** | Jaccard 文本相似度 | Embedding 语义相似度 + LLM Judge | OrthogonalCollisionDetector (NLI/SPO/数值三维度正交检测) |
| **事实理解** | 无 | Claim 提取（文本片段） | FactExtractor (SPO/NLI/数值/时间多维提取) |
| **分组策略** | 贪心最近邻 | 语义聚类 | Embedding层次聚类 + 事实桶 |
| **裁决机制** | 静态优先级 | LLM 智能判断 | 碰撞系数阈值 + LLM复核（可选） |
| **共识度计算** | 简单比例 | LLM判断结果 | 三维冲突比例加权 |

### 1.3 v3 碰撞检测算法流程概览

```
输入: List[ExtractedFact]
        │
        ▼
┌───────────────────┐
│ 1. 事实桶聚类      │  基于Embedding相似度将相似事实聚合
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 2. 正交碰撞检测    │  NLI/SPO/数值三维度独立检测
│   ┌─────────────┐ │
│   │ NLI维度     │ │  检测语义矛盾
│   ├─────────────┤ │
│   │ SPO维度     │ │  检测主谓宾冲突
│   ├─────────────┤ │
│   │ 数值维度    │ │  检测数值/时间冲突
│   └─────────────┘ │
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 3. 碰撞系数计算    │  α×nli + β×spo + γ×value
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 4. LLM复核（可选） │  碰撞系数≥阈值时触发
└───────────────────┘
        │
        ▼
   输出: List[TrustedFact]
```

## 2. 事实提取算法

### 2.1 数据结构

**ExtractedFact** 是碰撞检测的核心输入数据结构：

```python
@dataclass
class ExtractedFact:
    fact_id: str                       # 事实唯一标识
    statement: str                     # 事实陈述文本
    spo_triple: Optional[SPOTriple]   # SPO三元组（主语-谓词-宾语）
    nli_label: Optional[str]           # NLI标签（entailment/contradiction/neutral）
    numeric_values: List[NumericValue] # 数值信息列表
    datetime_values: List[DatetimeValue]# 时间信息列表
    confidence_score: float            # 置信度得分 (0-1)
    confidence_reason: str             # 置信度理由
    source_domain: Optional[str]        # 来源域名
    trust_score: float = 1000.0        # 信源可信度得分
```

### 2.2 SPO三元组

```python
@dataclass
class SPOTriple:
    subject: str    # 主语
    predicate: str  # 谓词
    object: str     # 宾语

    def __hash__(self):
        return hash((self.subject, self.predicate, self.object))
```

### 2.3 数值与时间

```python
@dataclass
class NumericValue:
    value: float                  # 数值
    unit: Optional[str] = None   # 单位
    original_text: Optional[str] = None

@dataclass
class DatetimeValue:
    value: str                    # 日期值
    parsed_value: Optional[str] = None
    original_text: Optional[str] = None
```

## 3. 事实桶聚类算法

### 3.1 聚类策略

事实桶聚类采用 **Embedding + 层次聚类** 策略，将语义相似的事实聚合到同一桶中：

```
输入: List[ExtractedFact]
        │
        ▼
┌───────────────────┐
│ 1. Embedding编码   │  将每条事实statement编码为向量
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 2. 相似度矩阵计算  │  余弦相似度计算
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 3. 层次聚类       │  scipy层次聚类，阈值0.75
└───────────────────┘
        │
        ▼
   输出: List[FactBucket]
```

### 3.2 聚类配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `similarity_threshold` | 0.75 | 层次聚类相似度阈值 |
| `use_hierarchical` | True | 是否使用层次聚类（False则用贪心） |

### 3.3 事实桶结构

```python
@dataclass
class FactBucket:
    bucket_id: str                      # 桶唯一标识
    facts: List[ExtractedFact]          # 桶内事实列表
    cluster_embedding: List[float]      # 聚类中心向量
```

## 4. 正交碰撞检测算法

### 4.1 架构概述

OrthogonalCollisionDetector 从三个正交维度检测碰撞：

```
                    ┌─────────────────────────────┐
                    │  OrthogonalCollisionDetector │
                    └─────────────────────────────┘
                                     │
        ┌────────────────────────────┼────────────────────────────┐
        │                            │                            │
        ▼                            ▼                            ▼
┌───────────────────┐    ┌───────────────────┐    ┌───────────────────┐
│  NLICollision     │    │  SPOCollision     │    │  ValueCollision   │
│  Detector         │    │  Detector         │    │  Detector         │
│                   │    │                   │    │                   │
│  • 语义矛盾检测   │    │  • 主语匹配       │    │  • 数值冲突检测   │
│  • 蕴含/矛盾/中立 │    │  • 谓词相反检测   │    │  • 日期冲突检测   │
│  • 否定词分析     │    │  • 宾语冲突检测   │    │  • 相对差异阈值   │
└───────────────────┘    └───────────────────┘    └───────────────────┘
        │                            │                            │
        ▼                            ▼                            ▼
   nli_conflict_ratio         spo_conflict_ratio          value_conflict_ratio
        │                            │                            │
        └────────────────────────────┼────────────────────────────┘
                                      │
                                      ▼
                           collision_coefficient
                           = α×nli + β×spo + γ×value
```

### 4.2 配置参数

```python
@dataclass
class OrthogonalCollisionConfig:
    alpha: float = 0.4              # NLI维度权重
    beta: float = 0.3              # SPO维度权重
    gamma: float = 0.3             # 数值维度权重
    threshold: float = 0.3         # 碰撞系数阈值
    nli_threshold: float = 0.5     # NLI冲突阈值
    spo_similarity_threshold: float = 0.8  # SPO相似度阈值
    numeric_threshold: float = 0.2 # 数值相对差异阈值
    datetime_threshold_days: int = 1  # 日期差异天数阈值
```

## 5. NLI碰撞检测器

### 5.1 检测逻辑

NLICollisionDetector 检测两个事实之间的语义矛盾关系：

```
检测规则:
1. 如果 fact_a.nli_label == CONTRADICTION 且 fact_b.nli_label == CONTRADICTION → 矛盾
2. 如果一个为ENTAILMENT，另一个为CONTRADICTION → 矛盾
3. 如果两个陈述的否定词分布相似但语义相反 → 触发语义矛盾检测
```

### 5.2 否定词分析

```python
negation_keywords = [
    "不是", "非", "没", "无", "未", "别", "否",      # 中文否定词
    "not", "no", "never", "none", "neither",         # 英文否定词
    "false", "wrong", "incorrect", "deny", "reject"  # 否定概念
]
```

当两个陈述都包含否定词，且通过 `difflib.SequenceMatcher` 计算的相似度 > 0.3 时，判定为语义矛盾。

### 5.3 冲突比例计算

```
nli_conflict_ratio = 矛盾对数量 / 总配对数量
```

其中总配对数量 = C(n, 2)，n为桶内事实数量。

### 5.4 核心实现

```python
def detect_nli_collision(self, facts: List[ExtractedFact]) -> float:
    if len(facts) < 2:
        return 0.0

    contradictions = 0
    total_pairs = 0

    for i in range(len(facts)):
        for j in range(i + 1, len(facts)):
            if self._is_contradiction(facts[i], facts[j]):
                contradictions += 1
            total_pairs += 1

    return contradictions / total_pairs if total_pairs > 0 else 0.0
```

## 6. SPO碰撞检测器

### 6.1 检测逻辑

SPOCollisionDetector 基于主谓宾三元组结构检测冲突：

```
检测条件（同时满足）:
1. 主语匹配（相同实体或相似度 >= similarity_threshold）
2. 谓词相反 或 宾语冲突
```

### 6.2 谓词相反判定

```python
opposite_pairs = [
    ("是", "不是"), ("在", "不在"), ("有", "没有"),
    ("能", "不能"), ("会", "不会"), ("支持", "反对"),
    ("赞成", "反对"), ("承认", "否认"), ("确认", "否认"),
    ("认可", "否定"), ("包含", "不包含"), ("属于", "不属于"),
    ("出生于", "死于"), ("结婚", "离婚"),
    ("上升", "下降"), ("增加", "减少"), ("买入", "卖出"),
    ("正面", "负面"), ("喜欢", "讨厌"), ("同意", "不同意"),
]
```

### 6.3 宾语冲突判定

1. **数值/布尔冲突**：两个宾语都是数值但值不同，或一个是"是"另一个是"否"
2. **反义词对**：如("真","假")、("对","错")、("true","false")
3. **相似度低于阈值**：diffflib相似度 < 0.8

### 6.4 核心实现

```python
def detect_spo_collision(self, facts: List[ExtractedFact]) -> float:
    spo_groups = self._group_by_subject(facts)

    conflicts = 0
    total_comparisons = 0

    for subject, group_facts in spo_groups.items():
        for i in range(len(group_facts)):
            for j in range(i + 1, len(group_facts)):
                if self._has_spo_conflict(group_facts[i], group_facts[j]):
                    conflicts += 1
                total_comparisons += 1

    return conflicts / total_comparisons if total_comparisons > 0 else 0.0
```

## 7. 数值碰撞检测器

### 7.1 检测逻辑

ValueCollisionDetector 检测数值和时间信息上的冲突：

```
数值冲突:
1. 按上下文关键词分组（增长、收入、价格等）
2. 比较同一上下文下两个数值的相对差异
3. 相对差异 > numeric_threshold (默认0.2) → 冲突

时间冲突:
1. 按年份分组
2. 计算日期差值
3. 差值 > datetime_threshold_days (默认1天) → 冲突
```

### 7.2 相对差异计算

```python
def _values_conflict(self, num_a: NumericValue, num_b: NumericValue) -> bool:
    larger = max(abs(val_a), abs(val_b))
    if larger > 0:
        relative_diff = abs(val_a - val_b) / larger
        if relative_diff > self.numeric_threshold:
            return True
    return val_a != val_b
```

### 7.3 单位兼容性

```python
compatible_groups = [
    {"km", "m", "cm", "mm", "mile", "foot", "inch"},  # 长度单位
    {"kg", "g", "mg", "lb", "pound", "ton"},            # 重量单位
    {"°C", "°F", "℃", "℉"},                           # 温度单位
    {"%", "percent", "percentage"},                    # 百分比
    {"年", "月", "日", "year", "month", "day"},        # 时间单位
]
```

## 8. 碰撞系数计算

### 8.1 公式定义

```
collision_coefficient = α × nli_conflict_ratio
                      + β × spo_conflict_ratio
                      + γ × value_conflict_ratio

其中 α = 0.4, β = 0.3, γ = 0.3
```

### 8.2 冲突事实判定

```python
def _get_all_conflicting_facts(self, facts: List[ExtractedFact]) -> List[ExtractedFact]:
    nli_conflict_ids = {f.fact_id for f in self.nli_detector.get_contradiction_facts(facts)}
    spo_conflict_ids = {f.fact_id for f in self.spo_detector.get_conflicting_facts(facts)}
    value_conflict_ids = {f.fact_id for f in self.value_detector.get_conflicting_facts(facts)}

    all_conflict_ids = nli_conflict_ids | spo_conflict_ids | value_conflict_ids
    return [f for f in facts if f.fact_id in all_conflict_ids]
```

### 8.3 碰撞结果结构

```python
@dataclass
class CollisionResult:
    bucket_id: str                      # 桶ID
    collision_coefficient: float        # 碰撞系数 (0-1)
    conflicting_facts: List[ExtractedFact]   # 冲突事实列表
    supporting_facts: List[ExtractedFact]     # 支持事实列表
    needs_llm_review: bool              # 是否需要LLM复核
    llm_review_result: Optional[str]    # LLM复核结果
    nli_conflict_ratio: float           # NLI冲突比例
    spo_conflict_ratio: float           # SPO冲突比例
    value_conflict_ratio: float         # 数值冲突比例
```

def _parse_verification_status(self, fact_id: str, llm_review_result: str) -> str:
    if "可信" in llm_review_result or "verified" in llm_review_result.lower():
        return "verified"
    if "存疑" in llm_review_result or "disputed" in llm_review_result.lower():

## 11. 算法复杂度分析

### 11.1 时间复杂度

| 阶段 | 时间复杂度 | 说明 |
|------|----------|------|
| 事实提取 | O(n) | 每条内容处理一次 |
| Embedding编码 | O(n) | 批量编码n条事实 |
| 事实桶聚类 | O(n²) | 相似度矩阵计算 |
| NLI碰撞检测 | O(m²) | 每个桶内m条事实配对 |
| SPO碰撞检测 | O(m²) | 每个桶内m条事实配对 |
| 数值碰撞检测 | O(m²) | 每个桶内m条事实配对 |

**总体时间复杂度：O(n² + k×m²)**
其中 n 为总事实数，k 为桶数量，m 为每桶平均事实数（通常 m << n）

### 11.2 空间复杂度

| 数据结构 | 空间复杂度 | 说明 |
|----------|----------|------|
| ExtractedFact列表 | O(n) | 存储n条事实 |
| Embedding向量 | O(n×d) | d为向量维度 |
| 相似度矩阵 | O(m²) | 每桶一个 |
| 聚类结果 | O(n) | 每条事实属于一个桶 |

**总体空间复杂度：O(n×d + n)**

## 12. v2到v3关键改进

### 12.1 从文本相似度到结构化事实

| v2 | v3 |
|----|----|
| Claim（文本片段，前200字符） | ExtractedFact（结构化，含SPO/NLI/数值/时间） |
| Jaccard/Embedding相似度 | NLI语义矛盾检测 |
| 单一相似度阈值 | 三维正交碰撞系数 |

### 12.2 从单维碰撞到正交检测

```
v2碰撞检测:
  claim1 ≈ claim2? (相似度 > 0.6)

v3碰撞检测:
  NLI维度: entailment/contradiction/neutral
  SPO维度: 主语匹配 + (谓词相反 OR 宾语冲突)
  数值维度: 相对差异 > 20%
         ↓
  collision_coefficient = 0.4×nli + 0.3×spo + 0.3×value
```

### 12.3 从贪心聚类到层次聚类

| v2 | v3 |
|----|----|
| 贪心最近邻聚类 | Embedding + scipy层次聚类 |
| O(n²)无优化 | 层次聚类自动确定最优分组 |
| 固定阈值0.6 | 可配置阈值0.75 |
