# 核心筛选算法技术文档

## 1. 概述

### 1.1 背景

在可信联网搜索系统中，信源筛选是确保返回结果可信度的第一道防线。系统需要对搜索结果按照信源质量进行分层过滤，区分高可信来源与低可信来源。

信源筛选算法（Source Classifier）通过规则匹配和模式推断，将搜索结果分为白名单（WHITE）、灰名单（GRAY）、黑名单（BLACK）三个层级，为后续的碰撞解决和摘要生成提供基础。

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| **SourceClassifier** | 信源分类器，核心筛选组件 |
| **WhitelistManager** | 白名单管理器 |
| **BlacklistManager** | 黑名单管理器 |
| **Classification** | 分层结果枚举，包含 WHITE/GRAY/BLACK |
| **SourceType** | 信源类型枚举，包含 OFFICIAL/MEDIA/KOL/INDIVIDUAL |
| **SourceLevel** | 信源层级枚举，包含 NATIONAL/PROVINCIAL/MUNICIPAL/LOCAL |
| **Rule Matching** | 规则匹配，判断信源是否满足特定规则 |

### 1.3 筛选流程概览

```
输入: List[SearchResult]
        │
        ▼
┌───────────────────┐
│ 1. 黑名单检查      │  优先检查是否命中黑名单规则
└───────────────────┘
        │
        ├──命中──→ BLACK (直接返回)
        │
        no
        ▼
┌───────────────────┐
│ 2. 白名单检查      │  检查是否命中白名单规则
└───────────────────┘
        │
        ├──命中──→ WHITE
        │
        no
        ▼
┌───────────────────┐
│ 3. 信源类型推断    │  推断信源类型 (OFFICIAL/KOL/MEDIA/INDIVIDUAL)
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ 4. 信源层级推断    │  推断信源层级 (NATIONAL/PROVINCIAL/MUNICIPAL/LOCAL)
└───────────────────┘
        │
        ▼
      GRAY (默认)
        │
        ▼
   输出: Classification (WHITE/GRAY/BLACK)
```

## 2. 分类优先级算法

### 2.1 优先级规则

分类优先级遵循 **BLACK > WHITE > GRAY** 的判断顺序：

```
优先级顺序:
1. BLACK (最高优先级) - 明确不可信的来源
2. WHITE - 明确可信的来源
3. GRAY (默认) - 既不在黑名单也不在白名单的来源
```

### 2.2 分类决策伪代码

```python
def classify(result: SearchResult) -> Classification:
    # Step 1: 优先检查黑名单
    if _is_blacklisted(result):
        return Classification.BLACK

    # Step 2: 检查白名单
    if _is_whitelisted(result):
        return Classification.WHITE

    # Step 3: 默认灰名单
    return Classification.GRAY
```

### 2.3 设计理由

**为什么 BLACK 优先级最高？**

黑名单代表已知的不可靠来源或违规信源，一旦命中应立即过滤。即使该信源恰好也在白名单中（例如配置错误），也应该以黑名单为准，确保安全性。

**为什么使用 GRAY 作为默认值？**

对于无法明确判断可信度的信源，默认归入灰名单。灰名单信源在碰撞解决时仍有机会被采纳，但优先级低于白名单信源。

## 3. 规则匹配算法

### 3.1 三种匹配模式

规则匹配支持三种模式，实现代码统一封装在 `_match_rule()` 方法中：

#### 3.1.1 精确域名匹配

```python
if "domain" in rule:
    if domain == rule["domain"].lower():
        return True
```

**示例：**
```yaml
- domain: "www.gov.cn"
  # 仅匹配 www.gov.cn
```

#### 3.1.2 域名后缀匹配

```python
if "domain_suffix" in rule:
    suffix = rule["domain_suffix"].lower()
    if domain.endswith(suffix):
        return True
```

**示例：**
```yaml
- domain_suffix: ".gov.cn"
  # 匹配所有以 .gov.cn 结尾的域名
  # 如: www.zj.gov.cn, www.gz.gov.cn
```

#### 3.1.3 正则表达式模式匹配

```python
if "domain_pattern" in rule:
    pattern = rule["domain_pattern"].lower().replace("*", ".*")
    if re.match(pattern, domain):
        return True
```

**示例：**
```yaml
- domain_pattern: "*.news.*"
  # 匹配任意以 .news. 开头或结尾的域名
  # 如: news.example.com, www.news.cn
```

### 3.2 匹配算法伪代码

```python
def _match_rule(result: SearchResult, rule: Dict) -> bool:
    domain = result.source_domain.lower()

    # 模式1: 精确域名匹配
    if "domain" in rule:
        if domain == rule["domain"].lower():
            return True

    # 模式2: 域名后缀匹配
    if "domain_suffix" in rule:
        suffix = rule["domain_suffix"].lower()
        if domain.endswith(suffix):
            return True

    # 模式3: 正则表达式匹配
    if "domain_pattern" in rule:
        pattern = rule["domain_pattern"].lower().replace("*", ".*")
        if re.match(pattern, domain):
            return True

    return False
```

## 4. 信源类型推断算法

### 4.1 信源类型定义

```python
class SourceType(Enum):
    OFFICIAL = "official"      # 政府机构、官方组织
    MEDIA = "media"            # 媒体、出版社
    KOL = "kol"               # 关键意见领袖
    INDIVIDUAL = "individual"  # 个人/自媒体
```

### 4.2 推断规则

信源类型推断使用**规则模式匹配**方法：

```
推断顺序:
1. 政府域名 (.gov.cn, .gov., .org.cn) → OFFICIAL
2. 有粉丝数且 >= 10000 → KOL
3. 媒体关键词匹配 → MEDIA
4. 默认 → INDIVIDUAL
```

### 4.3 推断算法实现

```python
def infer_source_type(result: SearchResult) -> SourceType:
    domain = result.source_domain.lower()

    # 规则1: 政府域名 → OFFICIAL
    government_suffixes = [".gov.cn", ".gov.", ".org.cn"]
    if any(domain.endswith(s) for s in government_suffixes):
        return SourceType.OFFICIAL

    # 规则2: KOL 判断 (有粉丝数且达到阈值)
    if result.粉丝数 and result.粉丝数 > 0:
        if result.粉丝数 >= self.kol_thresholds["small"]:  # 10000
            return SourceType.KOL

    # 规则3: 媒体关键词匹配 → MEDIA
    media_patterns = [
        "news", "press", "media", "journal",
        "xinhuanet", "people.com", "cctv", "cctv"
    ]
    if any(p in domain for p in media_patterns):
        return SourceType.MEDIA

    # 规则4: 默认 → INDIVIDUAL
    return SourceType.INDIVIDUAL
```

### 4.4 媒体关键词说明

| 关键词 | 示例域名 |
|--------|---------|
| news | news.sina.com.cn |
| press | press.cctv.com |
| media | media.cn |
| journal | journal.cn |
| xinhuanet | xinhuanet.com |
| people.com | people.com.cn |
| cctv | cctv.com |

### 4.5 KOL 粉丝阈值

| 等级 | 阈值 | 说明 |
|------|------|------|
| small | >= 10,000 | 小规模 KOL |
| medium | >= 100,000 | 中等规模 KOL |
| big | >= 1,000,000 | 大 V |

## 5. 信源层级推断算法

### 5.1 信源层级定义

```python
class SourceLevel(Enum):
    NATIONAL = "national"       # 国家级
    PROVINCIAL = "provincial"   # 省级
    MUNICIPAL = "municipal"     # 市级
    LOCAL = "local"            # 地方/县级
```

### 5.2 推断规则

信源层级推断主要基于**政府域名模式**：

```
层级推断规则:
1. 国家级: gov.cn, xinhuanet, people.com.cn, cctv.com, gov.hk, gov.tw, org.cn
2. 省级: [2位字母].gov.cn (如 ah.gov.cn, zj.gov.cn)
3. 市级: [3位及以上字母].gov.cn (如 gz.gov.cn)
4. 默认: LOCAL
```

### 5.3 推断算法实现

```python
def infer_source_level(result: SearchResult) -> SourceLevel:
    domain = result.source_domain.lower()

    # 规则1: 省级域名匹配 (*.ah.gov.cn, *.zj.gov.cn)
    # 正则: \.[a-z]{2}\.gov\.cn$
    provincial_pattern = r"\.[a-z]{2}\.gov\.cn$"
    if re.search(provincial_pattern, domain):
        return SourceLevel.PROVINCIAL

    # 规则2: 市级域名匹配 (*.gz.gov.cn, *.sz.gov.cn)
    # 正则: \.[a-z]{3,}\.gov\.cn$
    municipal_pattern = r"\.[a-z]{3,}\.gov\.cn$"
    if re.search(municipal_pattern, domain):
        return SourceLevel.MUNICIPAL

    # 规则3: 国家级模式匹配
    national_patterns = [
        "gov.cn", "xinhuanet", "people.com.cn", "cctv.com",
        "gov.hk", "gov.tw", "org.cn"
    ]
    if any(p in domain for p in national_patterns):
        return SourceLevel.NATIONAL

    # 规则4: 默认 → LOCAL
    return SourceLevel.LOCAL
```

### 5.4 正则表达式说明

**省级域名正则: `\.[a-z]{2}\.gov\.cn$`**

```
.ah.gov.cn      ✓ 匹配 (安徽)
.bj.gov.cn      ✓ 匹配 (北京)
.sh.gov.cn     ✓ 匹配 (上海)
.gz.gov.cn      ✗ 不匹配 (广州是3位)
```

**市级域名正则: `\.[a-z]{3,}\.gov\.cn$`**

```
.gz.gov.cn      ✓ 匹配 (广州)
.sz.gov.cn      ✓ 匹配 (深圳)
.heb.gov.cn     ✗ 不匹配 (河北是2位，省级)
```

## 6. 批量分类算法

### 6.1 批量分类接口

```python
def classify_results(
    self,
    results: List[SearchResult]
) -> Dict[str, List[SearchResult]]:
    """
    批量分类搜索结果

    Returns:
        {
            "white": [...],  # 白名单结果
            "gray": [...],   # 灰名单结果
            "black": [...]   # 黑名单结果
        }
    """
    classified = {
        "white": [],
        "gray": [],
        "black": []
    }

    for result in results:
        classification = self.classify(result)
        result.classification = classification
        classified[classification.value].append(result)

    return classified
```

### 6.2 批量分类流程

```
输入: List[SearchResult] [R1, R2, R3, ..., Rn]
        │
        ▼
┌─────────────────────────┐
│ 初始化结果容器           │
│ white=[], gray=[], black=[] │
└─────────────────────────┘
        │
        ▼
┌─────────────────────────┐
│ for result in results:  │
│   classify(result)      │
└─────────────────────────┘
        │
        ├──BLACK──→ black.append(result)
        ├──WHITE──→ white.append(result)
        └──GRAY───→ gray.append(result)
        │
        ▼
┌─────────────────────────┐
│ return {"white": [...], │
│         "gray": [...],   │
│         "black": [...]}  │
└─────────────────────────┘
```

## 7. 算法复杂度分析

### 7.1 时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|----------|------|
| 单结果分类 | O(m + n + k) | m=黑名单规则数, n=白名单规则数, k=媒体关键词数 |
| 批量分类 | O(t × (m + n + k)) | t=结果总数 |
| 类型推断 | O(1) | 固定模式匹配，无循环 |
| 层级推断 | O(1) | 固定正则匹配，无循环 |

**最坏情况：** 当黑名单和白名单都未命中时，需要遍历所有规则进行匹配。

### 7.2 空间复杂度

| 数据结构 | 空间复杂度 | 说明 |
|----------|----------|------|
| 规则列表 | O(m + n) | 存储黑白名单规则 |
| 分类结果 | O(t) | 存储 t 个结果及其分类 |

**总体空间复杂度：** O(m + n + t)

## 8. 独立组件设计

### 8.1 WhitelistManager

白名单管理器独立封装，提供 `is_whitelisted()` 方法：

```python
class WhitelistManager:
    def __init__(self, rules: List[Dict]):
        self.rules = rules

    def is_whitelisted(self, result: SearchResult) -> bool:
        for rule in self.rules:
            if self._match_rule(result, rule):
                return True
        return False

    def _match_rule(self, result: SearchResult, rule: Dict) -> bool:
        # 复用 SourceClassifier 的匹配逻辑
        domain = result.source_domain.lower()
        if "domain" in rule:
            if domain == rule["domain"].lower():
                return True
        if "domain_suffix" in rule:
            suffix = rule["domain_suffix"].lower()
            if domain.endswith(suffix):
                return True
        if "domain_pattern" in rule:
            pattern = rule["domain_pattern"].lower().replace("*", ".*")
            if re.match(pattern, domain):
                return True
        return False
```

### 8.3 RuleEngine

通用规则引擎，支持自定义分类规则：

```python
class RuleEngine:
    def __init__(self, rules: List[Dict] = None):
        self.rules = rules or []

    def match(self, result: SearchResult) -> Classification:
        for rule in self.rules:
            if self._match_rule(result, rule):
                return Classification(rule.get("classification", "gray"))
        return Classification.GRAY

    def _match_rule(self, result: SearchResult, rule: Dict) -> bool:
        domain = result.source_domain.lower()
        if "domain" in rule:
            return domain == rule["domain"].lower()
        if "domain_pattern" in rule:
            pattern = rule["domain_pattern"].lower().replace("*", ".*")
            return bool(re.match(pattern, domain))
        return False
```

## 10. 当前局限性与改进方向

### 10.1 当前局限性

| 局限性 | 说明 | 影响 |
|--------|------|------|
| **静态规则** | 规则需手动配置，无法动态更新 | 无法及时应对新出现的不可信信源 |
| **简单字符串匹配** | 无法处理 URL 重定向、子域名伪装 | 可能被绕过 |
| **无缓存机制** | 每次分类都重新匹配规则 | 重复查询效率低 |
| **单一域名维度** | 未考虑内容质量、历史记录等因素 | 分类粒度较粗 |

### 10.2 改进方向

#### 10.2.1 动态规则更新

**改进方案：** 引入规则热更新机制，支持运行时动态调整规则：

```python
def reload_rules(self):
    """重新加载规则配置"""
    self.whitelist = load_yaml("whitelist.yaml")
    self.blacklist = load_yaml("blacklist.yaml")
```

#### 10.2.2 URL 规范化

**改进方案：** 对 URL 进行预处理，去除跟踪参数、规范化格式：

```python
def _normalize_url(self, url: str) -> str:
    """规范化 URL"""
    # 去除 UTM 参数
    url = re.sub(r'\?utm_.*', '', url)
    # 提取主域名
    parsed = urlparse(url)
    return f"{parsed.netloc}"
```

#### 10.2.3 规则缓存

**改进方案：** 引入 LRU 缓存减少重复匹配：

```python

@lru_cache(maxsize=1024)

    """带缓存的规则匹配"""
    ...
```

#### 10.2.4 多维度评估

**改进方案：** 引入内容质量评分、历史可信度等多维度评估：

```python
def calculate_trust_score(self, result: SearchResult) -> float:
    """综合可信度评分"""
    domain_score = self._match_rules(result)  # 规则匹配得分
    content_score = self._analyze_content(result)  # 内容分析得分
    history_score = self._query_history(result.source_domain)  # 历史记录得分
    return 0.5 * domain_score + 0.3 * content_score + 0.2 * history_score
```
